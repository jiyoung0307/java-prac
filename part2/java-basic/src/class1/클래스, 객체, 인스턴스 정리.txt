* TIL : 클래스, 객체, 인스턴스
1. 클래스(Class)
- 클래스는 객체를 만들기 위한 틀(붕어빵 틀) 또는 설계도(자동차 설계도) 같은 것이다.
- 클래스 자체는 실체가 아니고, “이런 속성과 기능을 가진 애를 만들겠다” 하고 약속해둔 개념이다.

class Student {
    String name;
    int age;
    int grade;
}
→ Student 클래스는 "학생"이란 개념의 틀을 정의한 것일 뿐, 실제 학생은 아니다.

2. 객체(Object)
- 클래스라는 설계도를 이용해서 실제로 만들어진 실체가 객체다.
- 객체는 서로 독립된 상태와 값을 가진다.

Student student1 = new Student();
Student student2 = new Student();
→ student1과 student2는 같은 클래스(Student)로부터 만들어졌지만, 서로 다른 객체다.

3. 인스턴스(Instance)
- 특정 클래스에서 생성된 객체를 특별히 인스턴스라고 부른다.
- 보통 객체와 같은 뜻으로 쓰지만, “이 객체가 어떤 클래스에서 나왔는지”를 강조하고 싶을 때 사용한다.

→ student1은 Student 클래스의 인스턴스다.

4.객체 vs 인스턴스
- 둘 다 "클래스에서 실제로 만들어진 것"이라는 의미에서는 비슷하다.
- 차이점은 표현의 초점:
-- 객체(Object) → 그냥 “실체” 그 자체
-- 인스턴스(Instance) → “특정 클래스에서 나온 객체”라는 관계를 강조
- 정리: 모든 인스턴스는 객체이지만, 객체라고 해서 항상 인스턴스라고 표현하지는 않는다.
